## What is Super Protocol Testnet?

Testnet is an operational work-in-progress version of Super Protocol. It is continuously updated with new releases and functionality as the project moves towards Mainnet.  

The latest public release is Testnet Phase 5, launched in June 2024. It introduces the following major updates:

- **Open Marketplace**. Super Protocol Marketplace GUI no longer requires Testnet Access Token to view offers and check orders. The web-interface is available to anyone to browse even without the MetaMask extension installed. However, users still need Testnet credentials to get free testnet tokens and create orders.

- **Providers**. Super Protocol users can now register as _providers_ on Super Protocol. Providers can monetize their applications and datasets by offering them to other users. Since Super Protocol is still in the testnet stage, monetization is only possible with test tokens, as a demonstration of the functionality.

- **Users' solution and data offers**. Super Protocol users who registered as providers can create solution and data offers. It is possible to set offer restrictions, requirements, and prices. Hardware-based trusted execution environment (TEE) protects offered solutions and data. Customers cannot download, modify, or otherwise access the offer content even if they use it in their orders. In Testnet Phase 5, it is only possible to create solution and data offers. Super Protocol will add the ability to offer computational and storage resources in future releases.

- **Provider Tools**. This new CLI utility works alongside the existing Super Protocol tool called SPCTL. Provider Tools lets users register providers, set up new offers, and manage existing ones.

- **Updated documentation for providers**. The Super Protocol team encourages you to register a provider and create offers, even if only to try the new functionality. To guide you through the process, there is updated [documentation](/developers/cli_guides/providers_offers) covering the subject. If you are new to Super Protocol, start with the [basics](/developers/marketplace/first-steps/).

Super Protocol uses a continuous delivery approach, and the new Phase marks reaching several significant milestones. However, the team has constantly been adding many changes and updates.

## How to participate?

To participate in Super Protocol Testnet Phase 5, you need Testnet credentials.

- If you are a new user and have never applied, please fill out the [Testnet Participation Form](https://superprotocol.typeform.com/testnetm). 

- If you have previously applied for Testnet, then there is no need to reapply. You can use your existing Testnet Account.

- If you lost your Testnet invitation email from a previous Phase, contact Super Protocol on Discord or just apply again.

- If you have filled out the Testnet Participation Form, please wait for your turn! The Super Protocol team sends out invites daily, but it may take several days if the number of requests is high. You can ask  Community Managers on the Super Protocol [Discord server](https://discord.gg/superprotocol) about the status of your request. When your access is ready, you will receive a Testnet invitation email with Testnet credentials and detailed instructions.

## Support

If you have any issues or questions, contact Super Protocol on [Discord](https://discord.gg/superprotocol). The Community Managers will be happy to help you.
